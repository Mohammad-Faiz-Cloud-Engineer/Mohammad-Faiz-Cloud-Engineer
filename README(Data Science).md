## Data Science Principles

```yaml
Data Quality:
  - Clean, validate, and preprocess data rigorously
  - Never trust data without understanding its provenance
  - Document data lineage and transformations

Modeling:
  - Start simple, iterate fast
  - Always validate models on held-out data
  - Monitor for drift and degradation

Deployment:
  - Treat models as production software
  - Implement robust monitoring and alerting
  - version everything - data, code, and models

Insights:
  - Focus on business impact, not just accuracy metrics
  - Communicate findings clearly to non-technical stakeholders
  - Let data guide decisions, not confirm biases
```
